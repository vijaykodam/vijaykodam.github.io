Unlock Up to 40% AWS Cost Savings with Graviton!

There are multiple ways to cost optimize your applications running in AWS. One effective way to achieve this is by using Graviton-based instance types. Graviton processor-based instances are gaining popularity as more applications and AWS services now support them.

Amazon EC2 instances powered by AWS Graviton2 processors provide up to 40% better price performance compared to fifth-generation x86-based instances for a wide variety of workloads.

Climate-Friendly Computing with Graviton

Using Graviton also has climate benefits: it reduces your carbon footprint by using up to 60% less energy than comparable EC2 instances for the same performance.

By choosing Graviton-based instances over traditional x86 instances, you can unlock significant savings. Since Apple adopted ARM processors for their MacBooks, ARM has gained widespread support across operating systems and applications, ensuring seamless compatibility. AWS’s push for Graviton has further motivated the software industry to embrace ARM-based processors.

AWS has been steadily expanding Graviton support across its services, making it easier than ever to leverage cost savings with ARM-based processors.

Graviton Support Across AWS Managed Services

Here are some of the popular AWS managed services that support Graviton:

  • Amazon EKS
  • Amazon ECS
  • Amazon Aurora
  • Amazon RDS
  • AWS Lambda
  • Amazon OpenSearch Service

For AWS managed services where Graviton-based instances are supported, like EKS, you can create Graviton-based instances for the worker nodes. Refer to this link for a full list of supported AWS services.

  • Prometheus
  • Loki
  • Istio

Operating Systems Supporting Graviton

Several major operating systems also support Graviton, including Amazon Linux 2023, Ubuntu, RHEL, SUSE, Alpine Linux, and more.

Consider including Graviton-based instances in your workload, whether in your Kubernetes clusters, EC2 instances, Lambda functions, or databases.
